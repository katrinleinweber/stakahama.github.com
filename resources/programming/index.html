2 Classifications

To paraphrase/summarize Beating the Averages, not all languages are the same.

Tradeoffs:

  • high-level: save user time. choose based on
    • language expressiveness
    • libraries
  • low-level: save computational time

Paradigms:

  • procedural
  • object-oriented
  • functional
  • logic

Iimperative vs declarative

  • how to compute vs. what to compute

Static vs dynamic

  • Variable types are declared and remain invariant.

Strongly vs weakly typed

  • strict adherence to defined data type

General purpose languages vs domain-specific languages (DSLs)

  • DSLs often shorter because programs are constructed within context of a domain. For instance, split() in R implies splitting a table or vector. In Python, split() can be for strings or something else, so the context must be explicitly defined (through the namespace/object)
