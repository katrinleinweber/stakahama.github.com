Editors and Integrated Development Environments (IDEs)

This article summarizes the perceived value of a "simple" editor which allows you to program many languages with a minimal set of tools vs an IDE which allows you to program in a few languages with a set of specialized tools.

Many scientific software programs (Matlab, Igor, IDL) come with their own IDEs. A few independent IDEs are listed below (there are many, many more and each has its own advantages).

Language-agnostic editors/IDEs:

  • Emacs for everything. Editors provide part of the development environment are technically "integrated" with conventional unix tools like GNU make, the GNU debugger and the GNU compiler collection and other tools. Vim is also a popular editor; I personally use GNU Emacs.
  • Eclipse with plugins (Photran, Pydev, StatET, CDT). Some notes.
  • Microsoft Visual Studio. I used to use this in graduate school for MS Visual C++ and Digital Visual Fortran and was quite pleased with its user-friendliness. I was recommended the Kate IDE as a MS Visual Studio alternative for Linux, but it does not seem to be an equivalent replacement.

Language-specific IDEs:

  • RStudio for R

While knowing Emacs and Vim allow you to program in many languages very rapidly, the learning curve is quite steep, and relies on the programmer's toolbox (requires additional knowledge of GNU/Linux tools to be fully productive). An IDE like Eclipse is may be better suited for adoption by an experimentalist looking for a tool to help them program in a few languages.
