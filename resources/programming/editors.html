Editors and Integrated Development Environments (IDEs)

This article summarizes (very well) the perceived value of an "simple" editor which allows you to program many languages with a minimal set of tools, vs an IDE which allows you to program in a few languages with a set of specialized tools.

Many scientific software programs (Matlab, Igor, IDL) come with its own IDEs.

Language-agnostic editors/IDEs:

  • Emacs (or Vim) for everything. Editors provide part of the development environment are technically "integrated" with conventional unix tools like GNU make, the GNU debugger and the GNU compiler collection and other tools. I use Emacs
  • Eclipse with plugins (Photran, Pydev, StatET, CDT). Some notes.
  • Microsoft Visual Studio. I used to use this and was quite pleased with its user-friendliness. I was recommended to me Kate IDE as a MS Visual Studio alternative for Linux but it does not seem to be an equivalent replacement.

Language-specific IDEs:

  • RStudio for R

While knowing Emacs and Vim allow you to program in many languages very rapidly, the learning curve is quite steep, and relies on the programmer's toolbox (requires additional knowledge of GNU/Linux tools to be fully productive). An IDE like Eclipse is probably the better suited for adoption by an experimentalist looking for a tool to help them program in a few languages.
