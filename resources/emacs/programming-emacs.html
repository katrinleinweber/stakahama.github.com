Thoughts

  • The structure by which text-interaction is facilitated – through point, mark, and buffer rather than line-by-line – offers some ways to rethink text-processing (coming from batch-processing languages).
  • Program structure: no namespaces, dynamic scope – makes it difficult to write large, general programs (though people have written many good ones).
  • Dynamic scope: not too difficult to manage, though Common Lisp extensions provide lexical-let, lexical-let*, and labels. However, the value of the variable can be changed outside of the function (even using lexical-let).
