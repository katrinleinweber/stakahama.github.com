Laboratory notebooks were conceived when experiments could be described, recorded, and analyzed in a notebook entry. The sheer amount of data and complexity of analysis algorithms used nowadays makes it difficult to adapt the physical notebook to record the activities of modern science. I've looked into several electronic notebook alternatives, many of which specialize in chemical applications, but currently settled on a general-purpose note-taking tool – Emacs org-mode.

Emacs org-mode as logbook

Org-mode can more generally be used as an GTD tool, but simply for "logging" my activities the interactive features (TODO lists) are not as crucial (are we ever :DONE: in any case?) as searchability and ease of entry and representation of many different concepts. On this account, org-mode excels as it allows inclusion of tables, figures, equations, and ordered/unordered lists, with many options for exporting the entries. I can also leverage my existing proficiency in LaTeX for generating equations. LaTeX equation syntax is preferable to GUIs for selecting symbols, and MathML is difficult to code in directly (and not meant to be).

As a scientist, I want to leverage my existing proficiency in document preparation - e.g., \latex. I also prefer its (\latex) equation syntax to GUIs for selecting symbols, and MathML is not something you would code in directly.

LaTeX vs. HTML vs. …

LaTeX or HTML:

  • HTML - good for viewing/navigating on/over computer(s)
  • LaTeX documents are in printer-friendly format (desirable for generating hard-copies to be archived as "logbooks")

Org-mode implementations of LaTeX or HTML exporting functionality:

  • HTML
    • Equations are exported as images. Integration of MathML into org-mode export is uncertain. Edit: apparently not anymore.
    • Ignores double backslashes.
  • LaTeX
    • Formatting issues: need many newlines – setting parindent to 0 is one solution. In addition, some Emacs customizations are shown below.

Can also export to Emacs diary (which can be exported to iCal, GoogleCalendar…).

Implementation

  • Each calendar week corresponds to a separate file, named by the first day of the week (YYYY-MM-DD.org), and – if required – auxiliary materials (e.g., figures, calculations) are placed in a folder using the same naming convention (YYYY-MM-DD/ for the first day of the week).
  • Each heading is the (active) date generated by C-c ..
  • Each subheading is often the topic or project I am writing about (e.g., STXM, MD), and can also be tagged as such.
  • These files are organized into directories by year.

There are many ways to organize entries in a logbook or scientific notebook: sorting by topic, date, etc. In many examples that are given on how to keep a laboratory notebook, the test or hypothesis given can be completed in one to several days. In such cases it may be more prudent to keep these notebooks sorted by task as entries would be chronologically sorted, approximately. However, this is no longer the case as we are often required to work simultaneously on multiple tasks within multiple projects (with some degree of overlap among them), each of which can take much longer than days (and sometimes the duration and amount of content generated is not so easy to estimate), so sorting entries by date is the least ambiguous. Having said that, a tool like org-mode allows tagging by topic and entries can be later searched by these tags (e.g., STXM, MD) if/when desired (with agenda view: add to list with C-c [, invoke agenda view with C-c a m, and further manipulate with key commands).

Dates

C-c . to pull up calendar and insert date (default is current date). C-c ! to insert date and time. C-u C-c . and C-u C-c ! to insert inactive versions of the dates.

Quotes and code

See external page on literals.

For including sections of correspondence – enclose in #+BEGIN_EXAMPLE and #+END_EXAMPLE (each tag gets own line).

Examples on entering code: 

#+BEGIN_SRC python
  for i in range(2):
      print 'hello, world!'
#+END_SRC

#+BEGIN_SRC emacs-lisp
  (let (x)
    (dolist (x '(0 1))
      (print "hello, world!")))
#+END_SRC

Code can be in any language by emacs (mode exists) – emacs-lisp, python, r, matlab, etc. – if major-mode does not exist, will use generic code tags.

Emacs Lisp customizations

(add-to-list 'load-path "~/elisp/org-mode/lisp")
(require 'org-install)
(require 'org-latex)
;; The following lines are always needed.  Choose your own keys.
(add-to-list 'auto-mode-alist '("\\.org\\'" . org-mode))
(global-set-key "\C-cl" 'org-store-link)
(global-set-key "\C-ca" 'org-agenda)
(global-set-key "\C-cb" 'org-iswitchb)

;; mode-hook
(add-hook 'org-mode-hook
	  '(lambda()
	     (local-set-key [(shift f5)] 'org-export-as-latex)
	     (custom-set-variables
	      '(org-export-latex-date-format "")
	      '(org-export-latex-image-default-option "width=20em")
	      ))
)

;; redefine preamble for 'article' class
;; why use push: http://blog.plover.com/prog/elisp.html
(push '("article"
	"\\documentclass{article}
\\usepackage[utf8]{inputenc}
\\usepackage[T1]{fontenc}
\\usepackage{graphicx}
\\usepackage{hyperref}
\\usepackage{fullpage}
\\parindent 0pt"
	("\\section{%s}" . "\\section*{%s}")
	("\\subsection{%s}" . "\\subsection*{%s}")
	("\\subsubsection{%s}" . "\\subsubsection*{%s}")
	("\\paragraph{%s}" . "\\paragraph*{%s}")
	("\\subparagraph{%s}" . "\\subparagraph*{%s}"))
      org-export-latex-classes)

;; modify the header to say "Week of ..."  
;; turned on by default; can be unactivated for other documents
(defadvice org-export-latex-make-header 
  (before org-export-latex-make-header-format-title activate)
  "This advice function formats the title of the exported org
document to be read, \"Week of %s\" before it is
processed/formatted by org-export-latex-make-header. Disable with
the following lines:
(ad-disable-advice 'org-export-latex-make-header 'before
		   'org-export-latex-make-header-format-title)
(ad-activate 'org-export-latex-make-header)"
  (ad-set-arg 0 (format "Week of %s" (ad-get-arg 0))))

;; function to disable header modification
(defun disable--org-export-latex-make-header-format-title ()
  (interactive)
  (ad-disable-advice 'org-export-latex-make-header 'before
		     'org-export-latex-make-header-format-title)
  (ad-activate 'org-export-latex-make-header))

;; formatting options, primarily to remove "$" as math expression
;; 'matcher'.
;; from http://home.fnal.gov/~neilsen/notebook/emacs.html
(setq org-format-latex-options 
      '(:foreground default 
		    :background default 
		    :scale 1.0
		    :html-foreground "Black" 
		    :html-background "Transparent" 
		    :html-scale 1.0
		    :matchers ("begin" "$$" "\\(" "\\[")))

A few cons

  • File management (naming, dating) is user responsibility (though not entirely unlike physical log notebooks). Other "diary" software I've seen will often store entries in a single file. At some point this file will have to be separated chronologically; this method (using org-mode) will require that the separation be done front.
  • Static archiving not enforced.
  • Not notarized (unless cron set up to sync to a remote server daily?).
