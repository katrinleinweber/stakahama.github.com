Introduction

Short of writing our own editor from scratch, we can find one we like and add macros, functions, and packages to bring it closer to an editing/programming environment that suits our purpose. Bram Moolenaar, main author of Vim, writes: 

"[Y]ou should not try to learn every command an editor offers. That
would be a complete waste of time. Most people only need to learn 10
to 20 percent of the commands for their work. But it's a different set
of commands for everybody. It requires that you lean back now and
then, and wonder if there is some repetitive task that could be
automated. If you do a task only once, and don't expect having to do
it again, don't try to optimise it. But you probably realize you have
been repeating something several times in the last hour. Then search
the documentation for a command that can do it quicker. Or write a
macro to do it.  When it's a larger task, like lining out a specific
sort of text, you could look around in newsgroups or on the Internet
to see if somebody already solved it for you."

Emacs has been described as an execution environment for a lisp program designed for text-editing. In this interpretation, a minimalist language, emacs lisp, has been used to write and extend the program, consisting of libraries and extension packages. The standard distribution, as an amalgamation of the core language and its libraries, is quite large (in both size and functionality). As users, we can use these libraries and the underlying language to imrpove our productivity. Learning and customizing emacs to our purpose can have several benefits:

  • shorten the number of keystrokes for repetitive tasks, and
  • configure commands to reduce hand/finger strain.

To become a user of emacs, we can follow a series of tasks:

  • finding what functionality is available and learning the useful ones,
  • customizing variables,
  • defining aliases,
  • defining keybindings (or keyboard translations),
  • writing functions and macros (also, advising functions).

The following list of tasks describes the subset of emacs functionality with which I have become familiar:

  • scripting, interacting with interpreter (R, Python, Matlab)
  • editing and debugging code for compiled languages (C, Fortran, gdb)
  • writing papers and reports in LaTeX (and converting to MS Word-compatible format with LaTeX2rtf if necessary)
  • file manipulation (locally and remotely) (shell, dired, and emacs lisp for shell scripts)
  • maintaining a logbook with org-mode
  • editing files (text, HTML, etc.)
  • emacs lisp

These pages are mostly an aggregation of other users' content that I accumulated while learning to use emacs. This document is intended to provide a survey of the functionality and available libraries that I have considered and/or found useful.
